Roseify

UI Planner App Design Case Study

Purpose

To create a native mobile app for users who love to journal and plan out their futures in a more visual formal rather than all written out on just a list.

Roles

  • UI Designer
  • UX Researcher

Deliverables

  • Concept development
  • Competitive Analysis
  • User flow
  • Wireframes
  • Prototype
  • User interface

Duration

2 months

Challenges

  • To create a mobile app for iOS and Android following iOS's Human Interface Guidelines and Android's Material Design guidelines.
  • To design an app that implements friendly user navigation and recognizable design patterns.

Solutions

  • Implement a simplified interface format to stay in accordance with guidelines. Future iterations can allow room for animations and more features.
  • For friendly and familiar navigation, input native design patterns that include but are not limited to: date and time pickers, breadcrumb navigation, and data management.

Competitive Analysis

Three competitors were looked at through this process- being Notion, Structured, and Todoist. The importance of analyzing these competitors is to help determine what functions and features could be improved upon in ROSEify alongside what features could be used as inspiration.

Notion has really great sorting features that allows you to group tasks, journal entries, and you can make your own lists for different interests, so you're not confined to only a couple list options. But as a new user, there's so many sorting options available that it can be a little difficult to navigate.

Structured has a breadcrumb structure to it as you can sort your day into one tidy timeline. The visual layout is pleasing to the eye and the color choices work really well. The problems with Structured are that you can't create an untimed task and the first screen on the far left feels a little crowded. A drop down menu option would help alleviate this.

Todoist has really clear imagery on how to add tasks to to-do list. They use colorful icons to help differentiate actions and they utilize fun animation to make the app more memorable and unique. However, there's not a lot of customization options unlike Notion and Structured.

Once the mid-fidelity frames were fleshed out, a ton of changes occurred regarding user navigation and design layout. User testing allowed me to see the now-glaring problems within these frames. So, let's get into some of the issues.

One button says log in and one says sign up. Also suggestion for sign in and sign up buttons to be stacked instead of next to each other.

Headers need to be dragged to the edges of the screen. Also forward and back navigation needs to be reconsidered as it slows down the user.

There are too many forward and back arrows all around and it'll be important to narrow them down.

Icons need to be indicated with text until users are familiar with the app.

Top three looks more like buttons instead of text fields.
